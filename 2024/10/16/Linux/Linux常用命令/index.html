Linux 常用命令

2024-10-16 Linux Linux 0 评论

Linux 常用命令

常用

  • clear 清屏
  • history 查看历史命令记录
  • shutdown now 立马关机
  • shutdown -r 或者 reboot 重启
  • ps -ef 查看所有进程
  • ps -ef | grep 关键字 查看 java 关键字相关的进行
  • lsof -i:端口号 根据端口号查询进程
  • netstat -ano | findStr '端口号'
  • netstat -anp 查看端口情况
  • netstat -na | grep 端口号 查询某个端口
  • kill pid 杀死进程
  • kill -9 pid 彻底杀死进程
  • rpm -qa | grep http 模糊查询 linux 中的软件包
  • uname -a 查看linux 版本参数

目录 / 文件

操作

  • cd /目录路径/ 切换目录

    • cd .. 返回上一级目录

    • cd / 切换到根目录

  • ls 或者 ll：列出目录

  • pwd 显示当前所在目录

  • mkdir 目录名 创建一个目录

  • touch 文件名 创建一个文件

  • rmdir 目录名 删除一个目录

  • cp 目录名/文件名 新的目录地址 文件或目录

  • scp /目录名/文件名 角色@远程地址:远程目录名 scp -r 复制目录

  • mv 目录名/文件名 新的目录地址 文件或目录

  • rm -rf 目录名 递归删除这个目录下所有文件，包括目录

  • rm -f 文件名 删除文件

  • vi /路径/文件名 查看 / 编辑文件内容

    • i 编辑
    • esc + :wq 保存设置
    • esc + :q 退出

  • rpm -ivh rpm安装包 使用 rpm 安装

  • chmod 777 目录名或文件名 给目录或文件赋予最大操作权限

  • scp -P 2222 -r file_or_directory user@remote_host:/path/to/destination 从本地复制文件到远程主机

    scp -P 22 -r C:\Users\SseakomZ\Desktop\guard-adapter.jar root@172.16.7.210:/root/workspace/guard-adapter-prod/app-server/node-1/app

  • scp -P 2222 -r user@remote_host:/path/to/file_or_directory ./local_destination 从远程主机复制文件到本地

    • -P 指定端口号
    • -r 递归复制整个目录

  • chmod 755 omm

查看

  • cat xxx.log | grep -C 20 'xxx' 查看 xxx.log 指定内容前后 20行
  • cat -n <fileName> 查看文件并对行数进行编号
  • tail -f xxx.log 循环读取 xxx.log
  • find / -iname 'xxx' 在系统中查询zhi’ding

less

less <file>

按键/命令

  • [pagedown] ：向下翻页
  • [pageup] ：向上翻页
  • /字符串：在当前显示的内容（翻页进度位置），向下查找这个字符串关键字
  • ?字符串：向上查找字符串
  • n ：重复前一个查找，与/或?有关， 比如前一个命令是？表示向上查找，此时n会向上查找
  • N: 反向的重复前一个查找
  • g ：跳转到当前文件数据的第一行
  • G ：跳转到当前文件数据的最后一行
  • q ：退出当前文件的浏览

网络

  • systemctl restart network 重启网络服务
  • systemctl status network 查看网络状态
  • ifconfig 查看当前网络 IP
  • ip add/addr 查看 ip 配置
  • systemctl status firewalld.service 查看防火墙状态
  • systemctl start firewalld.service 开启防火墙
  • systemctl restart firewalld.service 重启防火墙
  • firewall-cmd --list-ports 查看已开放的端口
  • firewall-cmd --zone=public --add-port=端口号/tcp --permanent 开放指定端口
  • ssh root@ip 用 root 用户登录指定 ip 的服务器
  • ssh ip 用当前用户登录指定 ip 的服务器

服务

  • ps -ef | grep 服务名 查看服务是否存活
  • ps -aux |grep 服务名
  • ls -l /proc/pid 查看某个进程的相关信息，exe 表示绝对路径

资源

  • df -h 查看服务器磁盘空间
  • free 查看服务器内存使用情况
  • free -h 自动选择合适理解的单位展示内存使用情况
  • top 实时查看系统性能情况

安装

  • yum install xxx
  • rpm -ivh xxx``rpm -qa | grep xxx 查询某个套件是否已安装

配置 Linux 网络

  • vim /etc/sysconfig/network-scripts/ifcfg-ens33

  • ```shell
    TYPE=”Ethernet” #网络类型（通常是 Ethemet）
    PROXY_METHOD=”none”
    BROWSER_ONLY=”no”

    IPV4_FAILURE_FATAL=”no”

binlog

  • show variables like 'log_bin_basename'; 查询 binlog 完整路径

mysqlbinlog

  • mysqlbinlog <binlog 文件名> 查询某个 binlog 文件
    • mysqlbinlog --base64-output=DECODE-ROWS --verbose --database=apartment --table=system_org binlog.000003

RabbitMQ

  • whereis rabbitmq 查看 rabbitmq 安装地址
  • rabbitmqctl list_queues 查看 rabbitmq 消息队列列表
  • rabbitmqctl delete_queue 队列名称 删除指定的消息队列

MongoDB

  • mongodb://remoteuser:TrainP0ss!@10.72.8.248:27017,10.72.8.249:27017/ingredient 连接 mongodb
  • db 查看当前数据库
  • use 数据库名 切换数据库
  • help 查看帮助
  • db.auth( <username>, passwordPrompt() ) 权限认证密文
  • db.auth( <username>, <password> ) 权限认证明文
  • db.<collections>.find()..limit(<条数>)>.pretty() 查询某个集合的指定条数数据， pretty() 格式化数据结构

  • 启动 Java 程序，并开启远程调试

安装 Java（rpm）

  1. 将下载好的 jdk rpm 上传到服务器
  2. 检查是否有 jdk，rpm -qa |grep jdk
  3. 有的话先卸载 rpm -e –nodes $(rpm -qa |grep jdk)
  4. 安装完成后配置环境变量 文件：/etc/profile
    JAVA_HOME=/usr/java/jdk1.8.0_221-amd64（自己看自己usr/java目录下jdk信息）
    CLASSPATH=%JAVA_HOME%/lib:%JAVA_HOME%/jre/lib
    PATH=$PATH:$JAVA_HOME/bin:$JAVA_HOME/jre/bin
    export PATH CLASSPATH JAVA_HOME、
  5. 让新增的环境变量生效！ source /etc/profile

Linux 踩坑

依赖检测失败：rpmlib(PayloadIsZstd) <= 5.4.18-1 被 mysql-community-common-8.0.32-1.el9.x86_64 需要

解决方案：下错了 mysql 的安装包，下的是 linux 9 版本的，linux 7 不兼容。

Java 应用程序报端口占用，lsof -i:port 却找不到进程

解决方案：切换 root 再执行即可
