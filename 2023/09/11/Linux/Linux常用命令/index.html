Linux 常用命令

2023-09-11 Linux Linux 0 评论

Linux 常用命令

常用

  • clear 清屏
  • history 查看历史命令记录
  • shutdown now 立马关机
  • shutdown -r 或者 reboot 重启
  • ps -ef 查看所有进程
  • ps -ef | grep 关键字 查看 java 关键字相关的进行
  • lsof -i:端口号 根据端口号查询进程
  • netstat -ano | findStr '端口号'
  • netstat -anp 查看端口情况
  • netstat -na | grep 端口号 查询某个端口
  • kill pid 杀死进程
  • kill -9 pid 彻底杀死进程
  • rpm -qa | grep http 模糊查询 linux 中的软件包

目录 / 文件

操作

  • cd /目录路径/ 切换目录

    • cd .. 返回上一级目录

    • cd / 切换到根目录

  • ls 或者 ll：列出目录

  • pwd 显示当前所在目录

  • mkdir 目录名 创建一个目录

  • touch 文件名 创建一个文件

  • rmdir 目录名 删除一个目录

  • cp 目录名/文件名 新的目录地址 文件或目录

  • scp /目录名/文件名 角色@远程地址:远程目录名 scp -r 复制目录

  • mv 目录名/文件名 新的目录地址 文件或目录

  • rm -rf 目录名 递归删除这个目录下所有文件，包括目录

  • rm -f 文件名 删除文件

  • vi /路径/文件名 查看 / 编辑文件内容

    • i 编辑
    • esc + :wq 保存设置
    • esc + :q 退出

  • rpm -ivh rpm安装包 使用 rpm 安装

  • chmod 777 目录名或文件名 给目录或文件赋予最大操作权限

查看

  • cat xxx.log | grep -C 20 'xxx' 查看 xxx.log 指定内容前后 20行
  • tail -f xxx.log 循环读取 xxx.log

网络

  • systemctl restart network 重启网络服务
  • systemctl status network 查看网络状态
  • ifconfig 查看当前网络 IP
  • ip add/addr 查看 ip 配置
  • systemctl status firewalld.service 查看防火墙状态
  • systemctl start firewalld.service 开启防火墙
  • systemctl restart firewalld.service 重启防火墙
  • firewall-cmd --list-ports 查看已开放的端口
  • firewall-cmd --zone=public --add-port=端口号/tcp --permanent 开放指定端口
  • ssh root@ip 用 root 用户登录指定 ip 的服务器
  • ssh ip 用当前用户登录指定 ip 的服务器

服务

  • ps -ef | grep 服务名 查看服务是否存活
  • ls -l /proc/pid 查看某个进程的相关信息，exe 表示绝对路径

资源

  • free 查看服务器内存使用情况
  • free -h 自动选择合适理解的单位展示内存使用情况
  • top 实时查看系统性能情况

安装

  • yum install xxx
  • rpm -ivh xxx
  • rpm -qa | grep xxx 查询某个套件是否已安装

MySQL

  • systemctl start mysqld 启动 MySQL 服务
  • systemctl restart mysqld 重启 MySQL 服务
  • systemctl stop mysqld 停止 MySQL 服务
  • systemctl status mysqld 查看 MySQL 服务的状态
  • mysql -u用户名 -p密码 登录 MySQL

Java

  • 启动 Java 程序，并开启远程调试

Linux 踩坑

依赖检测失败：rpmlib(PayloadIsZstd) <= 5.4.18-1 被 mysql-community-common-8.0.32-1.el9.x86_64 需要

解决方案：下错了 mysql 的安装包，下的是 linux 9 版本的，linux 7 不兼容。

Java 应用程序报端口占用，lsof -i:port 却找不到进程

解决方案：切换 root 再执行即可
