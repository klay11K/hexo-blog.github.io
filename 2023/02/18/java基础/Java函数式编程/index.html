Java 函数式编程

什么是函数式编程

函数式编程是一种编程范式，它将计算视为函数的运算，并避免变化状态和可变数据。它是一种声明式编程范式，也就是说，编程是用表达式或声明而不是语句来完成的。

Lamda 表达式：(a,b) -> a+b

比起指令式编程，函数式编程更加强调程序执行的结果而非执行的过程，倡导利用若干简单的执行单元让计算结果不断渐进，逐层推导复杂的运算，而不是设计一个复杂的执行过程。

Lamda 表达式

箭头左边是参数列表，右边是函数体

方法引用 class::method

  • 静态方法引用

  • 参数方法引用

  • 实例方法引用

函数式编程的几个特点

  • 纯函数
    • 函数的执行没有副作用
    • 返回值仅依赖于输入参数
  • 高阶函数
    • 函数的参数可以是一个/多个函数
    • 函数的返回值可以是一个函数

内建的函数式接口

  • Function：表示一个函数（方法），它接受一个单一的参数并返回一个单一的值

  • Predicate：表示一个简单的函数，它只取一个值作为参数，并返回真或假

  • UnaryOperator：代表一个操作，它接受一个参数，并返回一个相同类型的参数。

  • BinaryOperator：代表接受两个参数并返回一个值的操作

  • Supplier：代表一个函数，它提供了某种类型值。

  • Consumer：代表一个接受参数而不返回任何值的函数

常见操作符

中间操作符

  • filter：过滤器，对数据进行过滤

  • map：对数据做一些额外的操作

  • peek：类似于 foreach，主要用于 debug

  • findAny：获取匹配的任意一个元素

  • findFirst：获取匹配的第一个元素

终端操作符

  • forEach：循环遍历

  • anyMatch：只要有任意一个元素匹配，返回 true，否则 false

  • noneMatch：没有匹配的返回 true，否则 false

聚合操作符

  • count：统计数据条数

  • min：取符合条件最小的值

  • max：取符合条件最大值

Optional

java 8 新特性，为了解决空指针异常而生

常见操作符

  • isPresent：如果存在，要执行的操作

  • ifPresentOrElse：如果存在，执行什么操作，如果不存在，执行什么操作

  • isEmpty：是否不存在

  • orElse：如果不存在，要执行的操作

  • orElseGet：如果不存在，要执行的操作

  • orElseThrow：如果不存在，抛出异常

  • or：如果不存在，返回一个 Optional 流的操作
